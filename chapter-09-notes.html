Part 02

คำนิยามของ Investment Property

Investment Property คืออสังหาริมทรัพย์ ซึ่งอาจเป็น

  • ที่ดิน
  • อาคาร
  • ส่วนของอาคาร
  • หรือทั้งที่ดินและอาคารรวมกัน

กิจการอาจถือทรัพย์นี้ในฐานะ

  • เจ้าของ
  • หรือผู้เช่าที่ถือครองในรูปสินทรัพย์สิทธิการใช้

จุดสำคัญคือวัตถุประสงค์ของการถือครองต้องเป็นอย่างใดอย่างหนึ่งต่อไปนี้

  • เพื่อหารายได้ค่าเช่า (Earn rentals)
  • เพื่อรอรับมูลค่าที่เพิ่มขึ้น (Capital appreciation)
  • หรือทั้งสองอย่าง
DEFAULT

สิ่งที่ไม่เข้าคำนิยาม

ถ้าถือครองเพื่อวัตถุประสงค์ต่อไปนี้ จะไม่ใช่ Investment Property

  • ใช้ในการผลิต
  • ใช้จัดหาสินค้าหรือบริการ
  • ใช้เพื่อการบริหารงาน
  • มีไว้เพื่อขายตามปกติของธุรกิจ

สรุปสั้นที่สุด:

  • ถ้าทรัพย์ “ทำเงินด้วยตัวมันเอง” จากค่าเช่าหรือราคาที่ขึ้น มักเป็น Investment Property
  • ถ้าทรัพย์เป็นส่วนหนึ่งของการดำเนินงานหลักของธุรกิจ มักไม่ใช่ Investment Property

Part 03

Owner-occupied property คืออะไร

Owner-occupied property คืออสังหาริมทรัพย์ที่กิจการถือไว้เพื่อ

  • ใช้ในการผลิต
  • ใช้จัดหาสินค้าและบริการ
  • ใช้เป็นสำนักงานเพื่อการบริหาร

หัวใจสำคัญคือ แหล่งที่มาของกระแสเงินสด

  • ถ้าเป็น Investment Property กระแสเงินสดจะมาจากตัวทรัพย์โดยตรง
  • ถ้าเป็น Owner-occupied property กระแสเงินสดจะมาจากกิจกรรมทางธุรกิจที่ใช้ทรัพย์นั้นเป็นส่วนประกอบ

ตัวอย่าง:

  • โรงงานผลิตเสื้อผ้าให้รายได้จากการขายเสื้อผ้า ไม่ได้มาจากตัวอาคารโรงงานโดยตรง

ดังนั้น ทรัพย์ที่ใช้งานเองจะไปอยู่ภายใต้มาตรฐานของ PPE

Part 04

Operating Lease vs Finance Lease ในมุมผู้ให้เช่า

ประเด็น Operating Lease Finance Lease
เนื้อหาทางเศรษฐกิจ ให้เช่าใช้พื้นที่/สินทรัพย์ คล้ายการให้สินเชื่อหรือขายผ่อน
ความเสี่ยงและผลตอบแทน ส่วนใหญ่ยังอยู่กับผู้ให้เช่า โอนไปยังผู้เช่าเกือบทั้งหมด
การจัดประเภทสินทรัพย์ ยังแสดงตัวทรัพย์อยู่ ตัดตัวทรัพย์ออกและรับรู้เป็นลูกหนี้
รายได้ที่รับรู้ รายได้ค่าเช่า รายได้ดอกเบี้ย + ตัดเงินต้น
ความเชื่อมโยงกับ Investment Property ใช่ ถ้าทรัพย์ถือไว้เพื่อรับค่าเช่า ไม่ใช่ เพราะไม่ใช่การถือทรัพย์เพื่อรับค่าเช่าในสาระอีกต่อไป
DEFAULT

อธิบายเพิ่ม

Operating Lease

  • ผู้ให้เช่ายังรับความเสี่ยงหลักของทรัพย์อยู่
  • ถ้าทรัพย์เสียหาย ราคาขึ้น หรือต้องซ่อมใหญ่ ผู้ให้เช่ายังเป็นฝ่ายรับผล
  • จึงยังถือว่าทรัพย์อยู่กับผู้ให้เช่า และหากเข้าเงื่อนไขก็เป็น Investment Property

Finance Lease

  • ลักษณะสัญญามักยาวมาก
  • ครอบคลุมอายุการใช้งานส่วนใหญ่ของสินทรัพย์
  • ทำให้ในเชิงสาระความเสี่ยงและผลประโยชน์หลักถูกโอนไปยังผู้เช่า

ดังนั้น ในงบของผู้ให้เช่า จะไม่ได้ถืออาคารเป็น Investment Property ต่อ

3.2 ที่ดินที่ยังไม่ได้กำหนดการใช้งานในอนาคตอย่างชัดเจน

เหตุผล:

  • หากกิจการยังไม่ตัดสินใจแน่ชัดว่าจะใช้เองหรือขาย

จึงตีความให้เป็น Investment Property

DEFAULT

3.3 อาคารที่กิจการเป็นเจ้าของและปล่อยเช่าแบบสัญญาเช่าดำเนินงาน

(A building owned by the entity and leased out under operating leases)

เหตุผล:

  • รายได้หลักจากทรัพย์มาจากค่าเช่า
  • กระแสเงินสดไม่ได้เกิดจากการผลิตสินค้าหรือให้บริการหลักของกิจการ

จึงเป็นตัวอย่างตรงของ Earn rentals

DEFAULT

3.4 อาคารว่างที่ถือไว้เพื่อเตรียมปล่อยเช่า

(A building that is vacant but is held to be leased out under operating leases)

เหตุผล:

  • แม้ยังไม่มีผู้เช่าในปัจจุบัน
  • แต่เจตนาการถือครองยังเป็นการสร้างรายได้ค่าเช่า

DEFAULT

3.5 อสังหาริมทรัพย์ที่กำลังก่อสร้างหรือพัฒนาเพื่อใช้เป็น Investment Property ในอนาคต

เหตุผล:

  • แม้จะยังสร้างไม่เสร็จ
  • แต่จุดหมายปลายทางของทรัพย์ถูกกำหนดแล้วว่าจะใช้เพื่อหารายได้ค่าเช่าหรือเก็งกำไร

  • เมื่อทำ Finance lease แล้ว ในเชิงสาระกิจการไม่ได้ถือทรัพย์เพื่อรับค่าเช่าแบบปกติอีกต่อไป

  • รายได้ที่รับรู้ต่อไปจะเป็น รายได้ดอกเบี้ย และการตัดเงินต้น ไม่ใช่ รายได้ค่าเช่า

Part 35

วงจรชีวิต Investment Property (IP) สรุปจบในหน้าเดียว

ใช้ทวนทั้งบทแบบมองเป็นลำดับตั้งแต่คัดกรองรายการ จนถึงการจำหน่ายและการเปิดเผยข้อมูล โดยคง logic หลักที่มักออกสอบไว้ครบในหน้าเดียว

PHASE 1

คัดกรองจุดยืน (ใช่ หรือ ไม่ใช่ IP?)

  • ใช่ IP: ถือไว้เพื่อรับค่าเช่า หรือ รอราคาขึ้น เช่น อพาร์ตเมนต์ หรือที่ดินว่างเปล่า
  • ไม่ใช่ IP: ใช้งานเอง หรือมีบริการเสริมเยอะ จัดเป็น PPE เช่น โรงแรม หรือออฟฟิศบริษัท
  • ไม่ใช่ IP: มีไว้เพื่อขายตามปกติธุรกิจ จัดเป็น Inventory เช่น หมู่บ้านจัดสรร
  • ไม่ใช่ IP: ถ้าปล่อยเช่าแบบโอนความเสี่ยงไปแล้ว เป็น Finance Lease และตัดตึกออกจากงบ
PHASE 2

วันแรกที่เข้าบ้าน (Initial Recognition)

กฎเหล็ก: วันแรกต้องบันทึกด้วย ราคาทุน (Cost) เสมอ

PHASE 3

การดูแลระหว่างทาง (Subsequent Costs)

  • เปลี่ยนชิ้นใหญ่ (Replacement): บวกทุนใหม่เข้าไป แล้วตัดมูลค่าบัญชีของเก่าทิ้ง
  • ซ่อมแซมรายวัน (Day-to-day servicing): รับรู้เป็น ค่าใช้จ่าย (Expense) ของงวด ห้ามบวกเข้าตึก
PHASE 4

ประเมินค่าสิ้นปี (Subsequent Measurement)

ต้องเลือกใช้วิธีใดวิธีหนึ่ง และบังคับใช้กับ IP ทุกชิ้นทั้งกลุ่ม

  • สาย Cost Model: คิดค่าเสื่อมราคาปกติ แต่ยังต้องหา Fair Value มาเปิดเผยในหมายเหตุประกอบงบเสมอ
  • สาย Fair Value Model: ห้ามคิดค่าเสื่อมราคาเด็ดขาด และส่วนต่างราคาตลาดที่ขึ้นหรือลงให้รับรู้เข้า P&L ทันที
PHASE 5

เปลี่ยนใจย้ายหมวด (Transfers)

กฎการโอน: จะโอนได้ต้องมี การเปลี่ยนการใช้งานจริง + มีหลักฐาน เท่านั้น แค่ผู้บริหารคิดเปลี่ยนใจเองยังโอนไม่ได้

จุดสลบปราบเซียน: โอน PPE เปลี่ยนเป็น IP (Fair Value)
  • ก่อนโอน: ต้องตีราคาแบบทิ้งทวนด้วยกฎของ PPE ก่อน ถ้าราคาขึ้นให้เอากำไรไปไว้ใน OCI / Revaluation surplus
  • หลังโอน: เมื่อกลายเป็น IP แล้ว ราคาที่ขึ้นหรือลงหลังจากนั้นจึงค่อยวิ่งเข้า P&L
PHASE 6

ขายทิ้ง / พังทลาย (Disposals)

สูตรคำนวณ: เงินที่ได้สุทธิ (Proceeds) - มูลค่าบัญชี (Carrying amount) = กำไร/ขาดทุน

ส่วนต่างนี้ให้รับรู้เข้า P&L ทันที

PHASE 7

เล่าให้นักลงทุนฟัง (Disclosure)

  • ต้องเปิดเผยนโยบายที่ใช้ว่าเป็น Cost หรือ Fair Value และบอกมูลค่ายุติธรรม
  • ต้องแยก Rental income และ Direct operating expenses ให้ชัด
  • ต้องแยกค่าใช้จ่ายของ ตึกที่มีคนเช่า ออกจาก ตึกที่ว่างเปล่า ด้วย
